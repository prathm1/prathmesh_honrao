<p>01.07.26</p><p>Problem I am solving today - Do we develop a solution for migration based on agentic ai, where every task is done by the agents, or do we use agentic ai software development to develop a python tool to then solve the migration problem ?</p><p></p><p>Thoughts so far - </p><p>-&gt; It depends on what the end goal is, do you want to setup a migration factory to regularly migrate code, or do you want to have a one time migration ? </p><p>-&gt; How important is deterministic behaviour for your results. For code translation, deterministic parsers make sense. For Document generation, business rules extraction, using AI agents makes sense</p><p>-&gt; Also depends on your token usage policy. Reusable tools, which only are updated with the help of ai when something breaks or needs an upgrade tend to use lower tokens in the long run, than using an agentic solution thats using llm for each and every task</p><p>🤔</p><p></p>
