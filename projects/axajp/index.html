Mainframe Discovery for Global Insurance in Japan

Duration: 3 months

Role: Lead Discovery Consultant

Technologies: COBOL, DB2 z/OS, VSAM, JCL, Stored Procedures, Oracle Cloud (OCI), AWS Sizing, Bash, Excel Macros

Overview

As part of a strategic partnership, I led the mainframe application discovery and mining project for a major insurance provider in Japan — a legacy-rich enterprise with decades of COBOL and DB2-based systems.

Our goals:

  • Understand the current application landscape
  • Audit both source code and database layers
  • Quantify modernization readiness
  • Propose a robust, cloud-aligned target architecture

Analysis Scope:

  • COBOL source code
  • Complexity metrics
  • Database DDLs
  • Stored Procedures
  • Business logic in exits
  • Application structure and integration points

Key Findings

Area Observation Impact
COBOL Codebase ~80% of programs contained unused or duplicated logic Increased complexity, performance degradation, migration bloat
Incremental Coding Style Programmers appended new logic rather than editing existing code Massive technical debt; bloated modules with layered legacy logic
Commenting Out Logic Rollback strategy relied on commenting out new code Inactive code cluttered the base, confused static analysis
Stored Procedures Many SPs performed business logic + embedded dynamic SQL Refactoring needed into Java service layers
Database Design Redundant indexes, absence of constraints, naming inconsistencies Suggested normalization, indexing, and refactoring strategies

Approach

Static Analysis

  • Total LoC
  • Nested IF/EVALUATE levels
  • Dead/duplicate code patterns
  • Copybook usage statistics

Manual Review

  • JCL job flows
  • Schedulers, FTP pipelines
  • Custom assembler logic

Mapping Created

  • Module ↔ Copybook
  • Program ↔ Table
  • JCL ↔ Batch Schedule

Target Architecture Recommendations

Target Cloud: AWS (EC2, Aurora PostgreSQL, Lambda, CloudWatch)

Tooling:

  • Code: Innowake for refactoring COBOL to Java
  • Data: AWS DMS for bulk loads, Oracle GoldenGate for sync
  • Infra-as-Code: Terraform templates
  • Sizing: Service-by-service cost and config estimates using existing LPAR CPU/memory metrics

Lessons Learned

  • Dead code = dead weight. It needs early and automated detection.
  • Cultural practices (e.g., non-destructive coding) drastically impact modernization.
  • Stored procedures need thorough behavioral mapping — they often contain the real logic.
  • Always give the client a visual blueprint. Most don't know the full picture of their estate.
  • Discovery is both technical and strategic. The report should enable decision-making.
