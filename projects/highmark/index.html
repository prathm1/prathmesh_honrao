Mainframe Application Development for a U.S.-based Medicaid Insurance Provider

Duration: ~2 years

Role: COBOL Developer (Application Development – Batch Systems)

Client: Major U.S. Medicaid & Health Insurance Provider

Employer: Accenture (Hyderabad Delivery Center)

Tech Stack: COBOL, JCL, DB2 for z/OS, VSAM, Endevor, File-Aid

Overview

This project marked my first professional engagement in mainframe technologies, where I was involved in the end-to-end development of a critical batch-processing application for a U.S.-based Medicaid and health insurance giant.

The client managed large volumes of member, provider, and claims data across multiple U.S. states. The goal of this initiative was to automate key Medicaid claims processing functions and reduce the dependency on manual workflows, improving compliance and throughput.

I was part of a distributed Agile team and took ownership of several use cases from design to implementation.

Sample Use Case Developed

Use Case: Automate claim rejection for eligibility mismatches

  • Input: Member file (VSAM), claims extract (flat file)
  • Output: Rejected claims (report file) + DB update to logging table
  • COBOL module: Performed file matching, rule checks, and DB logging
  • JCL: Defined multi-step batch job with restart logic and override capability

Lessons Learned

  • Modularity is key: Designing reusable, well-structured COBOL subroutines makes maintenance easier.
  • Data accuracy matters: Even a small misalignment in flat files or a wrong key can cause batch failures.
  • Mainframe batch windows are sacred: Understanding job priorities, scheduling dependencies, and restart logic is crucial.
  • JCL skills are underrated: Effective JCL control can drastically reduce failure rates.
  • Legacy ≠ Obsolete: COBOL systems can be robust and high-performance when well designed.
