Mainframe Discovery for a Leading Danish Insurance Company

Duration: 4 months

Role: Lead ‚Äì Database Discovery & Modernization Strategy

Technologies: DB2 z/OS, VSAM, JCL, COBOL, Assembler, Azure SQL, Java, Microservices, Event-driven Architecture

Overview

A major insurance provider in Denmark engaged our team to perform a comprehensive discovery of their large-scale mainframe installation. The client's estate spanned multiple LPARs, hosting mission-critical insurance applications across COBOL, DB2, and VSAM systems.

Objectives:

  • Audit the existing system landscape (applications, databases, batch jobs)
  • Identify modernization blockers
  • Suggest a future-ready architecture on cloud

Key Focus Areas

  • Complete DB2 DDL inventory and data model evaluation
  • Deep analysis of stored procedures, UDFs, and assembler exits
  • Understanding and documenting batch job chains and dependencies
  • Mapping COBOL modules to underlying data access logic
  • Developing a modernization plan aligned with Azure SQL and event-driven principles

Findings Summary

Area Observation Impact
DB2 Stored Procedures Heavy use of business logic inside SPs and UDFs Required modularization for replatforming into microservices
Assembler Exits Custom DB2 intercept logic embedded at various access points Non-portable and undocumented; required decommission or refactoring plan
Batch Processing Nightly batch jobs transferring high-volume data through chains of JCL jobs Needed transformation into event-driven, cloud-native equivalents
VSAM & Legacy Files Thousands of files read/written via COBOL/FTP Phased migration proposed using API gateways and object storage

Proposed Target Architecture

Core Components

  • Database: Azure SQL PaaS as the target for DB2 workloads
  • Data Movement: Azure Data Factory pipelines + SSIS integration
  • Application Layer: Java-based microservices hosted on Azure App Services
  • Inter-process Communication: Azure Service Bus for batch-to-event translation
  • Monitoring & Audit: Azure Monitor and Log Analytics

Migration Strategy

  • Modularize SPs and logic-heavy DB objects
  • Decommission assembler exits after behavior replication
  • Replace file-based FTP flows with APIs and storage containers
