Mainframe Modernization POC for a Global Automotive Manufacturer

Duration: 2-3 months

Role: Technical Lead – Database Migration, Coordination, Documentation

Client: Global Automobile Brand (Germany)

Scope: 2 critical mainframe applications with business logic in COBOL stored procedures

Focus Areas: COBOL-to-Java migration, Innowake tooling, Dockerized DB2 environment, AI-accelerated documentation

Overview

This Proof of Concept (POC) engagement aimed to demonstrate the feasibility of modernizing two mainframe applications owned by a global automotive giant. These applications relied heavily on COBOL-based stored procedures in DB2 z/OS, where the business logic was deeply embedded in the database layer.

Objectives:

  • Refactor COBOL stored procedures into Java using Innowake tooling
  • Set up a testing environment using Docker-based DB2 LUW containers
  • Validate functional correctness through unit tests, inserts, and updates
  • Deliver a comprehensive discovery and mining report detailing the process, observations, and next steps

Migration Approach

Step Description
1 Extracted COBOL stored procedures (STPs) from DB2 z/OS
2 Applied Innowake 1:1 transformation to generate Java equivalents
3 Developed thin Java wrappers to maintain interface compatibility
4 Deployed the transformed Java code in a local DB2 LUW Docker image
5 Executed unit and functional test cases to validate correctness and integration

Outcomes & Results

  • ✅ Full functional parity was achieved between COBOL STPs and the generated Java versions.
  • ✅ Client engineers performed independent UAT and confirmed that data insert/update operations were successful.
  • ✅ The Java code (aka Jobol) was seen as highly readable and close to their internal Google-style standards.
  • ✅ Demonstrated the feasibility of setting up a factory model for high-volume STP migration.
  • ✅ Generated client trust in the transformation pipeline's speed, reliability, and maintainability.

My Contributions

  • Acted as end-to-end technical bridge between the client, Deloitte India delivery team, and the Innowake engineering team.
  • Led the DB2 LUW environment setup via Docker on local and client infrastructure for testing and demonstration.
  • Provided in-depth DB2 technical support, explaining runtime behavior, limitations, and conversion quirks.
  • Created a customized discovery and mining report:
    • Captured the current COBOL logic and the transformed Java logic
    • Detailed test results, mappings, and tool behavior
    • Included actionable next steps and improvement areas
  • Leveraged AI tools (ChatGPT) to accelerate documentation, summarize transformations, and present findings clearly.
  • Led the AI walkthrough session during client demo, showcasing future roadmap opportunities.

Lessons Learned

  • COBOL stored procedures that encapsulate business logic are prime candidates for 1:1 refactoring using Innowake or similar tooling.
  • Dockerized DB2 LUW environments offer a lightweight, replicable setup for local transformation validation.
  • Java wrappers need to maintain interface parity with calling programs to avoid downstream changes.
  • Client confidence builds quickly when they are given transformed code that works within 2–3 weeks.
  • AI tools can drastically improve speed, structure, and impact of technical documentation.

Skills Demonstrated

  • Java transformation logic and testing

  • Report writing and AI-assisted documentation

POC Checklist

  • ✅ Assess where the real business logic lives — inside COBOL or stored procedures
  • ✅ Plan for 1:1 migration tooling that supports procedural DB logic
  • ✅ Use containerized DB2 LUW setup to isolate issues and speed up feedback
  • ✅ Validate insert/update/test cases early with client
  • ✅ Generate mapping tables and traceability docs alongside code
  • ✅ Keep a lean scope, but deliver a visibly working result
  • ✅ Use AI tools for discovery, report generation, and documentation acceleration
  • ✅ Leave client with a clear understanding of next steps and expansion path
